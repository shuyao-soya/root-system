# 词根图谱 · 跨设备同步版 · 部署说明

## 这次改了什么（国内可访问性专项修复）

你反馈国内的家人打不开网站，排查后有三个原因，这版全部处理了：

1. **Vercel 默认域名在国内被 DNS 污染** —— 需要你自己绑定一个自定义域名（见下面"第 0 步"）
2. **Google Fonts 在国内基本连不上** —— 已移除，改用系统自带字体，视觉效果接近，不再依赖外网字体
3. **加载 Supabase 库用的 esm.sh 在国内不稳定** —— 已改成从本地文件加载，你需要做一次性的手动下载（见"第 3.5 步"）

同时也带上了之前的功能：放射状词根图、同步口令、录入新词、导出/导入备份、恢复默认词表、离线可用（service worker 已修复缓存 bug，见上一次的说明）。

---

## 第 0 步（新增，解决国内访问问题）：绑定自定义域名 + 中国友好 CNAME

Vercel 的默认域名 `xxx.vercel.app` 在中国大陆几乎必定打不开，这一步是解决你爸妈访问问题的关键：

1. 买一个域名（任意平台都行：腾讯云、阿里云、Namecheap……不需要备案，内容还是托管在 Vercel）
2. Vercel 项目 → **Settings → Domains** → 绑定这个域名
3. 去域名的 DNS 解析后台，加一条 **CNAME** 记录，主机记录填 `@` 或 `www`（看你想用哪个），**记录值填**：
   ```
   cname-china.vercel-dns.com
   ```
   注意：不要用 Vercel 默认自动分配的那个 CNAME 目标，一定要手动改成这个
4. 如果还是连不上，可以再加一条 **A 记录**，值填 `76.223.126.88` 做兜底
5. 部署完成后，用「站长工具」一类的多地 Ping 测试网站测一下你的新域名，确认国内各地运营商能否连通（不需要你人在国内也能测）

**如实说明**：这个方法目前是社区公认最有效的方案，但不是 100% 保证——国内网络管控会变化，个别地区/时段仍可能连不上。绑完之后务必实测。

## 第 1-3 步：Supabase 部分（和之前一样，如果你已经建过表，跳过）

1. https://supabase.com 注册项目
2. SQL Editor 里跑 `supabase-setup.sql`
3. Project Settings → API，复制 **Project URL** 和 **anon public key**

## 第 3.5 步（新增）：下载 supabase.js 到本地

因为要去掉 esm.sh 这个国内不稳定的依赖，Supabase 的库文件需要你手动下载一次，放进项目文件夹：

1. 浏览器打开这个地址：
   ```
   https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2/dist/umd/supabase.js
   ```
2. 右键 → "另存为"，文件名存成 `supabase.js`
3. 把这个文件放到你项目文件夹里，和 `index.html` 同一级目录

（如果你熟悉命令行，也可以直接用 `curl -o supabase.js https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2/dist/umd/supabase.js`）

## 第 4 步：把 Supabase 配置填进代码

打开 `index.html`，找到：
```js
const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
```
换成你自己的值。

## 第 5 步：部署到 Vercel

这次要上传 **五个文件**（比之前多了 `supabase.js`）：
- `index.html`
- `manifest.json`
- `sw.js`
- `supabase.js`（新增，第 3.5 步下载的）
- （`supabase-setup.sql` 不用传，只是给你在 Supabase 后台跑的）

替换掉 Vercel 项目里的旧文件，重新部署。

## 第 6 步：清一次旧缓存

因为 `sw.js` 之前有个缓存 bug（已修复），已经打开过网站的设备（包括你自己的）需要手动清一次：
- 电脑：F12 → Application → Service Workers → Unregister，再强制刷新（Ctrl+Shift+R）
- 手机：把网站从主屏幕删掉重新添加，或清一下浏览器该网站的缓存

之后再重新部署，就不会有这个问题了（新版 `sw.js` 会自动拿最新版本）。

## 如果国内还是打不开，下一步能做什么

如果绑了自定义域名 + 中国友好 CNAME 之后，家人那边还是不行，说明问题可能出在 Supabase 本身（它托管在 AWS 海外节点，理论上国内能连，但个别网络环境可能慢或不稳）。到时候可以考虑的方向：
- 换一个对国内更友好的数据库服务（我可以帮你评估）
- 或者退一步：给爸妈提供一个"离线安装包"版本，定期手动导出/导入备份同步，不依赖实时连接

先测一下现在这版效果如何，再看要不要往这个方向走。

## 文件清单

| 文件 | 作用 |
|---|---|
| `index.html` | 网站主文件（这次已经帮你把 Supabase URL/Key 填好了，不用再改） |
| `manifest.json` | 让网站可以"添加到主屏幕"，这次接入了你的 `icon.svg` 作为应用图标 |
| `sw.js` | 离线可用 + 自动获取最新版本（v2，已修复缓存 bug） |
| `icon.svg` | 应用图标（用的是你上传给我的那份） |
| `supabase.js` | Supabase 客户端库（还需要你手动下载一次，见"第 3.5 步"） |
| `supabase-setup.sql` | 建表脚本，只需在 Supabase SQL Editor 里跑一次（如果之前跑过就不用再跑） |

**这次部署请把 Vercel 项目文件夹整个清空重传这 6 个文件**（`supabase-setup.sql` 除外，那个不用传，只在 Supabase 后台跑），不要和旧文件混着放——上次的问题就是新旧文件混用导致的。
